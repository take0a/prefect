Prefect Cloud には、安定性を確保するための制御機能が備わっています。

## API レート制限

<Note>
**Prefect Cloud のレート制限は変更される場合があります**

**API のレート制限は変更される場合があります**
レート制限に関するご質問は、Prefect サポート（[help@prefect.io](mailto:help@prefect.io)）までお問い合わせください。
</Note>

Prefect Cloud の API レート制限は、特定のエンドポイントに対して、一定期間内に単一のクライアントから送信できるリクエスト数を制限します。
`flow_runs`、`task_runs`、`flows` エンドポイントとそのサブルートは、以下のレート制限を受けます。

- Free アカウントの場合、1 分あたり 400 リクエスト
- Pro アカウントの場合、1 分あたり 2,000 リクエスト

これらのエンドポイントは、この制限に達した場合、適切な `Retry-After` ヘッダーを含む `429` レスポンスを返します。
クライアント側での再試行の処理方法とデフォルトの動作を変更する方法の詳細については、[ClientSettings](/v3/develop/settings-ref#clientsettings) を参照してください。

`logs` エンドポイントの制限は以下のとおりです。

- 無料アカウントの場合、1分あたり700件のログ
- Pro アカウントの場合、1分あたり10,000件のログ

この制限を超えた場合、エンドポイントは `429` レスポンスを返します。

## メタデータの保持期間

フロー実行、タスク実行、およびアーティファクトのメタデータは、Prefect Cloud アカウントの保持期間に従って保持されます。
この保持期間は、アカウントに属するすべてのワークスペースに適用されます。

保持期間とは、メタデータが作成されてから利用可能な日数です。
フロー実行とタスク実行の場合、保持期間は実行が[ターミナル状態](/v3/develop/manage-states/#state-types)に達した時点から計算されます。
サブフロー実行は、親フロー実行とは独立して保持されます。各サブフロー実行がターミナル状態に達した時点に基づいて削除されます。

カスタム保持期間が必要な場合は、[Prefect の営業チームにお問い合わせください](https://www.prefect.io/pricing)。
