---
title: APIキーを管理する
description: ローカル実行環境から Prefect Cloud にアクセスするための API キーを作成します。
---

APIキーを使用すると、Prefect Cloud を使用するためのローカル環境の認証が可能になります。

CLI から `prefect cloud login` を実行する場合、ブラウザ経由、または API キーを貼り付けて認証できます。

ブラウザ経由で認証すると、認証ページが表示されます。
接続を承認すると、CLI にリダイレクトされ、API キーがローカルの [Prefect プロファイル](/v3/develop/settings-and-profiles/) に保存されます。

APIキーを貼り付けて認証する場合は、まず Prefect Cloud UI で API キーを作成する必要があります。

## APIキーの作成

1. UIの左下にあるアカウントアイコンを選択します。
2. **APIキー**を選択します。
このページには、以前に生成されたキーのリストが表示され、APIキーの作成または削除が可能です。
3. **+**ボタンを選択して、新しいAPIキーを作成します。
キーの名前と有効期限を入力します。

<Warning>
API キーは生成後に UI で再度公開することはできないため、キーを安全な場所にコピーします。
</Warning>

## APIキーでPrefect Cloudにログインする

```bash
prefect cloud login -k '<my-api-key>'
```

あるいは、CLI が利用できない場合（例えば、リモートサーバーレス環境内で Prefect Cloud に接続する場合など）は、`PREFECT_API_KEY` 環境変数を設定してください。

詳細については、[Prefect Cloud への接続](/v3/manage/cloud/connect-to-cloud) を参照してください。

## サービスアカウント API キー (Pro) (Enterprise)

サービスアカウントは、Prefect Cloud [Pro および Enterprise プラン](https://www.prefect.io/pricing) の機能であり、ユーザーアカウントに関連付けられていない Prefect Cloud API キーを作成できます。

サービスアカウントは、ワーカーの実行やリモートインフラストラクチャでのフロー実行のための API アクセスを構成する際に便利です。
これらの環境でのフロー実行のイベントとログは、ユーザーではなくサービスアカウントに関連付けられます。
サービスアカウントを構成したり削除したりすることで、ユーザーアクセスを中断することなく API アクセスを管理または取り消すことができます。

詳細については、[サービスアカウント](/v3/manage/cloud/manage-users/service-accounts/) をご覧ください。
