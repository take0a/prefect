---
title: サービスアカウントを管理する
description: ワークスペースは、Prefect Cloud 内のフローとデプロイメントのための分離された環境です。
---

サービスアカウントを使用すると、ユーザーアカウントに関連付けられていない Prefect Cloud API キーを作成できます。
サービスアカウントは通常、リモートインフラストラクチャ上でワーカーを実行したり、デプロイフローを実行したりするための API アクセスを構成するために使用されます。

サービスアカウントは、以下の機能を持つ非ユーザーアカウントです。

- Prefect Cloud [API キー](/v3/manage/cloud/manage-users/api-keys/)
- [ロール](/v3/manage/cloud/manage-users/manage-roles/) と権限

サービスアカウントの認証情報を使用すると、[実行環境を構成](/v3/manage/cloud/connect-to-cloud/#configure-a-local-execution-environment)して、ユーザーが手動でその環境にログインすることなく、Prefect Cloud ワークスペースとやり取りできます。
サービスアカウントは、他のユーザーアカウントに影響を与えることなく、作成、ワークスペースへの追加、ロールの変更、削除を行うことができます。
サービスアカウントには、有効なワークスペースレベルのロールを適用できます。

サービスアカウントを表示、作成、または編集するには、[**サービスアカウント**] を選択します。

サービス アカウントはアカウント レベルで作成されますが、個々のワークスペースをサービス アカウントと共有できます。詳細については、[ワークスペースの共有](/v3/manage/cloud/workspaces/#workspace-sharing) をご覧ください。

<Tip>
**サービスアカウントの認証情報**

サービスアカウントを作成すると、Prefect Cloud はアカウント用の新しい API キーを作成し、実行環境用の API 設定コマンドを提供します。
これらの認証情報は、将来使用するために安全な場所に保存してください。
アクセス認証情報を紛失または漏洩した場合は、サービスアカウントページから再生成してください。
</Tip>

## サービスアカウントを作成します

**「サービスアカウント」ページで**+**アイコンを選択して、新しいサービスアカウントを作成します。以下の項目を設定します。

- サービスアカウント名。アカウント内で一意の名前である必要があります。
- 有効期限、または**無期限**オプション。

<Note>
**サービスアカウントのロール**

サービスアカウントは、アカウントのメンバーとしてのみ使用できます。
サービスアカウントを[ワークスペースに追加](/v3/manage/cloud/workspaces/#workspace-sharing)する際に、有効な_ワークスペースレベルの_ロールを適用できます。
</Note>

新しいサービス アカウントを作成するには、[**作成**] を選択します。

<Warning>
API キーは生成後に UI で再度公開することはできないため、キーを安全な場所にコピーします。
</Warning>

サービスアカウントのAPIキーと有効期限を変更するには、APIキーをローテーションします。
このページのサービスアカウント情報の左側にあるメニューから「**APIキーのローテーション**」を選択します。
必要に応じて、古いサービスアカウントキーを有効のままにする期間を設定できます。

サービスアカウントを削除するには、サービスアカウント情報の左側にあるメニューから「**削除**」を選択します。
