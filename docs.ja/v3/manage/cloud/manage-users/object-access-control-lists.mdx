Prefect Cloud の [エンタープライズプラン](https://www.prefect.io/pricing) では、オブジェクトレベルのアクセス制御リスト (ACL) を使用して、ワークスペース内の特定のユーザーとサービスアカウントへのアクセスを制限できます。
ACL は、ブロック、デプロイメント、ワークプールでサポートされています。

組織管理者とワークスペース所有者は、オブジェクトに移動して [**アクセスを管理**] をクリックすることで、アクセス制御リストを設定できます。
ACL が追加されると、ワークスペースロールを通じてオブジェクトにアクセスできるすべてのユーザーとサービスアカウントは、ACL に明示的に追加されていない場合、アクセスできなくなります。

<Note>
**ACL と可視性**

フロー実行、フロー、アーティファクトなど、アクセス制御リストによって管理されていないオブジェクトは、関連するブロックまたはデプロイメントがユーザーに対して制限されている場合でも、ワークスペース内のユーザーに表示されます。
</Note>

オブジェクト レベルのアクセス制御をサポートするためのオプションの詳細については、[Prefect Cloud プラン](https://www.prefect.io/pricing) を参照してください。

## ワークプールとデプロイメントの ACL 委任

デプロイメントは、権限チェックをワークプールに委任できます。この委任は次のように機能します。

1. ワークプールに ACL が設定されている場合、その ACL はワークプールを使用するすべてのデプロイメントに適用されます。
2. ワークプールに ACL が設定されていない場合、代わりに個々のデプロイメントの ACL が適用されます。

この委任システムにより、特に複数のデプロイメントで同じワークプールを使用する場合に、権限をより効率的に管理できます。
