---
title: シングルサインオンを構成する
description: Prefect Cloud ユーザー向けにシングル サインオン (SSO) を構成します。
---

Prefect Cloud の [エンタープライズプラン](https://www.prefect.io/pricing) では、チームの ID プロバイダーとのシングルサインオン (SSO) 統合が提供されます。
以下の機能をサポートする任意の ID プロバイダーと SSO 統合を設定できます:

- OIDC
- SAML 2.0

SSO を使用する場合、Prefect Cloud は ID プロバイダが管理するアカウントのパスワードを保存しません。
Prefect Cloud アカウントのメンバーは、ID プロバイダを使用してログインおよび認証します。

SSO 統合が設定されると、管理者以外のユーザーはアカウントリソースにアクセスする際に SSO プロバイダを介して認証する必要があります。

より多くのユーザー、ワークスペース、サービスアカウント、および SSO をサポートするためのオプションの詳細については、[Prefect Cloud プラン](https://www.prefect.io/pricing) をご覧ください。

## SSO の設定

アカウント内で「**SSO**」ページを選択し、ユーザーの SSO を有効にしてください。

ドメインでまだ SSO を有効にしていない場合は、Prefect Cloud で SSO を有効にするメールドメインを入力し、保存してください。

「**有効なドメイン**」の下にある「**ドメイン**」リストからドメインを選択し、「**リンクを生成**」を選択してください。
この手順により、ID プロバイダで SSO を設定するためのリンクが作成されます。

提供されたリンクを使用して「ID プロバイダ設定」ダッシュボードに移動し、ID プロバイダを選択して設定を続行してください。
ご利用のプロバイダがリストにない場合は、「SAML」または「Open ID Connect」をお試しください。

![Opening the Identity Provider Configuration dashboard.](/v3/img/ui/cloud-sso-dashboard.png)

SSO 構成が完了すると、ユーザーはアカウント リソースにアクセスするときに ID プロバイダーを通じて認証する必要があり、アプリケーション アクセスを完全に制御できるようになります。

## ディレクトリ同期

**ディレクトリ同期** は、アカウントのユーザーのプロビジョニングとデプロビジョニングを自動的に実行します。

プロビジョニングされたユーザーには基本的な「メンバー」ロールが付与され、そのロールに必要なすべてのリソースにアクセスできます。

アイデンティティプロバイダで Prefect Cloud アプリケーションからユーザーの割り当てが解除されると、そのユーザーは自動的に Prefect Cloud リソースにアクセスできなくなります。
これにより、IT チームは Prefect UI にサインインすることなく、Prefect Cloud へのアクセスを制御できます。

## SCIM プロビジョニング

エンタープライズプランでは、ユーザープロビジョニングのために SCIM にアクセスできます。
SSO タブから SCIM プロビジョニングを有効にすることができます。
