ワークスペースとは、Prefect Cloud 内でワークフローや関連情報のための独立した環境です。
たとえば、開発環境、ステージング環境、本番環境を分離したり、異なるチームを分離したりするために、個別のワークスペースを使用できます。

## ワークスペースを使い始めましょう

Prefect Cloud に初めてログインすると、独自のワークスペースを作成するように求められます。

アクセスできるすべてのワークスペースを表示するには、ナビゲーションサイドバーの上部にあるワークスペース名をクリックします。
アクセスできるすべてのワークスペースを表示するには、ナビゲーションサイドバーの上部にあるワークスペース名をクリックします。

利用可能なワークスペースのリストには、以下のものが含まれます。

* ご自身のアカウントのワークスペース。
* 管理者またはメンバー権限で招待されたアカウントのワークスペース。

ワークスペース内のユーザー権限は異なる場合があります。アカウント管理者は、ワークスペースレベルで役割と権限を割り当てることができます。

## ワークスペースを作成する

アカウント設定のドロップダウンで、「新しいワークスペースを作成」ボタンを選択し、新しいワークスペースを作成します。

以下の項目を設定します。

* ドロップダウンのアカウントメニューオプションから、**ワークスペース所有者** を選択します。
* **ワークスペース名** はアカウント内で一意である必要があります。
* ワークスペースの説明（任意）

**作成** を選択して新しいワークスペースを作成します。利用可能なワークスペースの数は、[Prefect Cloud プラン](https://www.prefect.io/pricing/) によって異なります。
追加のワークスペースまたはユーザーが必要な場合は、[料金](https://www.prefect.io/pricing/) をご覧ください。

## アクセス制御

Prefect Cloud Pro または Enterprise レベルのアカウントでは、ワークスペースオーナーは他のユーザーをメンバーとして招待したり、ワークスペースに [サービスアカウント](https://docs.prefect.io/ui/service-accounts/) をプロビジョニングしたりできます。
ワークスペースオーナーは、ユーザーに [ワークスペースロール](https://docs.prefect.io/ui/roles/) を割り当て、ワークスペース内でのユーザーの権限範囲を指定します。

ワークスペースオーナーは、**[ワークスペース] -> [メンバー]** を選択してメンバーを管理し、**[ワークスペース] -> [サービスアカウント]** を選択してワークスペースのサービスアカウントを管理します。

以前に招待されたユーザーやプロビジョニング済みのサービスアカウントがここに表示されます。

アカウントにユーザーを招待するには、**[メンバー +]** アイコンを選択します。既存のアカウントメンバーのリストから選択できます。

ユーザーのロールを選択します。これは、ワークスペース内でのユーザーの初期ロールです。

ワークスペースオーナーは、このロールをいつでも変更できます。

ワークスペースのメンバーを削除するには、このページのユーザー情報の右側にあるメニューから [**削除**] を選択します。

ワークスペースにサービスアカウントを追加するには、[**サービスアカウント +**] アイコンを選択します。
**名前**、**アカウントロール**、**有効期限**、およびオプションで**ワークスペース** を選択します。

サービスアカウントの API キーを更新、削除、またはローテーションするには、サービスアカウントの右側にあるメニューから既存のサービスアカウントのオプションを選択します。

## ワークスペースの移行

ワークスペースの移行を使用すると、既存のワークスペースをあるアカウントから別のアカウントに移動できます。

ワークスペースの移行では、ブロック、デプロイメント、オートメーション、ワークプール、ログなど、既存のワークスペース構成とフロー実行履歴が保持されます。

<Note>
**ワークスペースの移行権限**

ワークスペースの移行を開始または承認するには、管理者権限が必要です。

個人アカウント間でワークスペースの移行を開始するには、[support@prefect.io](mailto:support@prefect.io) までご連絡ください。
</Note>

サイドバーナビゲーションでワークスペース名をクリックし、「**ワークスペース設定**」を選択します。

ページ右上の3点メニューから、「**移行**」を選択します。

「**ワークスペースの移行**」ページの左側に移行先のワークスペースが表示されます。
右側で、ワークスペースの移行先アカウントを選択します。

<Tip>
**ワークスペースの移行によるアカウントへの影響**

ワークスペースの移行は、移行元アカウントと移行先アカウントのリソース使用量とコストに影響を与える可能性があります。

ワークスペースを移行すると、ユーザー、API キー、およびサービスアカウントはワークスペースにアクセスできなくなる可能性があります。

監査ログはワークスペース上のアクティビティを追跡しなくなります。
移行先アカウントのフロー実行保持期間外に終了したフロー実行は削除されます。
また、ワークスペースの新しい場所にアクセスするために、Prefect CLI プロファイルと実行環境の設定を更新する必要がある場合もあります。

移行先のアカウントで、移行されたワークスペースに対応するために新たな料金が発生する場合があります。
</Tip>

## ワークスペースを削除する

ワークスペースを削除すると、デプロイメント、フロー実行履歴、ワークプール、自動化など、そのワークスペースのすべてのデータが削除されます。

サイドバーナビゲーションでワークスペース名をクリックし、「**ワークスペース設定**」を選択します。

ページ右上の3点メニューから、「**削除**」を選択します。
