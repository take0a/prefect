Prefect Cloud は、オープンソースの Prefect サーバーの全機能に加え、追加機能を備えたホスト型ワークフローアプリケーションプラットフォームです。

- [Prefect Cloud への接続](/v3/manage/cloud/connect-to-cloud/) では、Prefect Cloud にアクセスするためのローカル実行環境の設定方法を説明します。

以下のページでは、Prefect Cloud の機能について詳しく説明します:

- [ワークスペースの管理](/v3/manage/cloud/workspaces/) では、アクティビティを整理するために分離された環境を使用する方法を説明します。

**アカウントの管理** サブセクションでは、Prefect Cloud アカウントに関する以下の側面について説明します:

- [ユーザーアカウントの管理](/v3/manage/cloud/manage-users/) では、Prefect Cloud の個人アカウントの使用方法について説明します。
- [サービスアカウントの管理](/v3/manage/cloud/manage-users/service-accounts/) では、ユーザーアカウントに関連付けられていない API キーを使用して、リモートインフラストラクチャ上でワーカーを実行したりフローを実行したりするための API アクセスを構成する方法について説明します。<span class="badge pro"></span> <span class="badge enterprise"></span>
- [チームの管理](/v3/manage/cloud/manage-users/manage-teams/) では、ユーザーグループをまとめて管理する方法を説明します。 <span class="badge enterprise"></span>
- [アカウントロールの管理](/v3/manage/cloud/manage-users/manage-roles/)では、ロールベースのアクセス制御（RBAC）の使用方法を説明します。RBACは、アカウントまたはワークスペース内で特定のアクティビティを実行するためのきめ細かな権限設定です。エンタープライズプランでは、特定の権限を持つカスタムロールを作成できます。
- [APIキーの管理](/v3/manage/cloud/manage-users/api-keys/)では、ローカル実行環境からPrefect Cloudへのアクセスを許可する方法について説明します。
- [シングルサインオンの設定](/v3/manage/cloud/manage-users/configure-sso/)では、OIDCおよびSAMLを使用したIDプロバイダーによる(SSO)認証統合のサポート方法について説明します。ディレクトリ同期とSCIMプロビジョニングも利用できます。 <span class="badge enterprise"></span>
- [Audit Cloud Activity](/v3/manage/cloud/manage-users/audit-logs/) は、Prefect がセキュリティとコンプライアンスを監視するためにユーザーアクティビティの記録を提供する仕組みを示しています。<span class="badge pro"></span> <span class="badge enterprise"></span>
- [Object Access Control List](/v3/manage/cloud/manage-users/object-access-control-lists/) (ACL) を使用すると、特権ユーザーはデプロイメントを制限し、個々のユーザーへのアクセスをブロックできます。<span class="badge enterprise"></span>
- [Secure access by IP](/v3/manage/cloud/manage-users/secure-access-by-ip-address/) は、IP アドレスで Prefect Cloud へのアクセスを制限する仕組みを示しています。 <span class="badge enterprise"></span>
- [PrivateLink によるセキュアアクセス](/v3/manage/cloud/manage-users/secure-access-by-private-link/) では、PrivateLink による Prefect Cloud へのアクセスを制限する方法を説明します。<span class="badge enterprise"></span>

このセクションの残りのPrefect Cloudページでは、Prefect Cloudのさまざまな側面について説明します:

- [レート制限とデータ保持](/v3/manage/cloud/rate-limits/)では、Prefect Cloudのレート制限とデータ保持について説明します。
- [Terraformプロバイダ](https://registry.terraform.io/providers/PrefectHQ/prefect/latest/docs/guides/getting-started)では、Infrastructure as Code用のPrefect Cloud Terraformプロバイダドキュメントへのリンクを示します。
- [Prefect Cloudのトラブルシューティング](/v3/manage/cloud/troubleshoot-cloud/)では、Prefect Cloudのトラブルシューティングに関する一般的な解決策を紹介します。

ドキュメント内の他の場所にある Prefect Cloud 関連のページには、以下が含まれます。

- [プッシュワークプール](/v3/deploy/infrastructure-examples/serverless/) を使用すると、ワーカーを実行せずにサーバーレスインフラストラクチャ上でフローを実行できます。
- [マネージドワークプール](/v3/deploy/infrastructure-examples/managed/) を使用すると、ワーカーを実行せずに Prefect インフラストラクチャ上でフローを実行できます。
- [Webhook](/v3/automate/events/webhook-triggers/) 。

Marvin AI によって有効化されたエラーサマリーは、「失敗」および「クラッシュ」したフロー実行のエラーログを実用的な情報に抽出します。
この機能を有効にするには、アカウントの**設定**ページにアクセスしてください。

[Prefect Cloud REST API](/v3/api-ref/rest-api/) は、Prefect クライアントと Prefect Cloud 間のオーケストレーションと監視のためのデータの送受信に使用されます。
Prefect のインタラクティブ Cloud REST API ドキュメントは、<a href="https://app.prefect.cloud/api/docs" target="_blank">https://app.prefect.cloud/api/docs</a> でご覧いただけます。


## Prefect Cloud を試す

Prefect Cloud を試すには、[app.prefect.cloud](https://app.prefect.cloud/) で無料アカウントを作成してください。
UI の手順に従って、Prefect Cloud で最初のワークフローを作成し、実行してください。
