---
title: Docker
description: Docker を使用して独自の Prefect サーバー インスタンスをセルフホストします。
---

コンテナ内でサーバーを実行するには、Prefect 公式 Docker イメージを使用できます。

Docker コンテナ内でサーバーを実行する場合は、以下の手順が必要です。

- サーバーのポートをローカルマシンに転送します (例: `-p 4200:4200`)。
- `prefect server start` コマンドの `--host` フラグを `0.0.0.0` に設定します。
- コンテナ内で Prefect を使用するには、API サーバーのアドレス (`PREFECT_API_URL`) を設定します。

例:

```bash
docker run -p 4200:4200 -d --rm prefecthq/prefect:3-latest -- prefect server start --host 0.0.0.0
```

<Tip>
`-d` フラグはコンテナをデタッチモード (つまりバックグラウンド) で実行し、`--rm` フラグはコンテナが停止したらそれを削除します。
</Tip>

詳細については、[ホスト ネットワークを使用したネットワーク](https://docs.docker.com/network/network-tutorial-host/)に関する Docker トピックを参照してください。
