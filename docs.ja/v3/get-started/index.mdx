Prefect は、Python 関数を最小限の手間で本番環境レベルのデータパイプラインに変換するオープンソースのオーケストレーションエンジンです。
DSL や複雑な設定ファイルを必要とせず、純粋な Python でワークフローを構築・スケジュールし、どこでも実行できます。
Prefect は、自動状態追跡、障害処理、リアルタイム監視など、面倒な処理をすぐに実行できます。

**なぜ？** 2018年、私たちは多くの方がよくご存知の問題を解決することを目指しました。それは、データワークフローのオーケストレーションを、歯を抜くような苦労（例えば、YAMLでDAGを記述すること）ではなく、もっと簡単に実現したいというものです。
Prefectは、Airflowのようなツールが対応に苦戦していた課題、つまり動的なワークフロー、最新のインフラストラクチャ、そして今日のデータパイプラインの複雑さに対処するために設計されました。
それ以来、Prefectはスタートアップ企業からFortune 100企業まで、あらゆる企業にとって頼りになるツールとなっています。
Prefectは、手間をかけずに本番環境対応のワークフローをリリースできるよう支援します。そのため、デバッグに費やす時間を減らし、構築に多くの時間を費やすことができます。

### 必須機能
<div className="t-full-width">
| Feature | Description |
|---------|-------------|
| **Pythonic** | ネイティブPythonでワークフローを記述できます。DSL、YAML、特別な構文は不要です。型ヒント、async/await、最新のPythonパターンをフルサポート。既存のIDE、デバッガー、テストツールもご利用いただけます。 |
| **State & Recovery** | 成功、失敗、再試行の状態を追跡する堅牢な状態管理。中断された実行を最後の成功時点から再開し、高負荷な計算をキャッシュすることで不要な再処理を回避します。 |
| **Flexible & Portable Execution** | フローをローカルで開始することで開発が容易になり、単一のプロセスからコンテナ、Kubernetes、クラウドサービスまで、ベンダーに縛られることなく、どこにでもデプロイできます。インフラストラクチャは構成だけでなくコードによって定義されるため、環境の拡張や変更も容易です。 |
| **Event-Driven** | スケジュール、外部イベント、またはAPI経由でフローをトリガーします。フローを一時停止して、人による介入や承認を求めます。状態、条件、または任意のカスタムロジックに基づいてフローを連結します。 |
| **Dynamic Runtime** | 実際のデータや条件に基づいて、実行時に動的にタスクを作成します。実行中に新しいタスクやブランチを簡単に生成し、真のデータ駆動型ワークフローを実現します。 |
| **Modern UI** | 直感的なインターフェースで、フロー実行のリアルタイム監視、ログ記録、状態追跡が可能です。フローを実行してUIを開くだけで、依存関係グラフとDAGを自動的に表示できます。 |
| **CI/CD First** | 通常のPythonコードと同様にフローをテストおよびシミュレートし、開発中に迅速なフィードバックを提供します。既存のCI/CDパイプラインにシームレスに統合し、テストとデプロイメントを自動化します。 |
</div>

## Prefect のミニ歴史

**2018-2021:** 私たちの物語は、ワークフローオーケストレーションはPythonで実現すべきというアイデアを発表した2018年に始まります。
Daskのような分散ツールにインスピレーションを受け、創業者であるJeremiah Lowin（Apache AirflowのPMCメンバー）の経験を基に、タスクとフロー用のシンプルなPythonデコレータをベースとしたシステムを開発しました。
しかし、Prefectを真に特別なものにしたのは、タスクマッピングの導入でした。これは後に動的実行機能の基盤となる機能であり、最終的には他のオーケストレーションSDKにも模倣されました。


**2022:** 現実世界のワークフローは、必ずしも整然とした事前に計画されたDAG構造に収まるわけではないことを認識した時点で、Prefect 2.0のリリースは必然となりました。例えば、ワークフローの分岐をスキップするなど、実行時情報に基づいてジョブ定義を更新する必要がある場合があります。
そこで、ワークフローをDAGとして明示的に記述するという重要な制約を取り除き、if/else条件文やwhileループなど、PythonをPythonたらしめるあらゆる要素をネイティブのPython制御フローとして完全に採用しました。


**2023-present:** 2024年にリリースされたPrefect 3.0では、イベントと自動化のバックエンドをオープンソース化することで、これらの動的パターンを全面的に採用しました。これにより、ユーザーはイベントドリブンのワークフローをネイティブに表現し、その実行に対する可観測性を高めることができます。
Prefect 3.0はパフォーマンスの飛躍的な向上も実現し、Prefectの実行時オーバーヘッドを最大90%改善しました。
